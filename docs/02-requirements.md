# 需求沉淀

## 核心业务目标

你想要的不是一个通用投顾机器人，而是一套围绕“债市温度”做资产配置调整的自动化系统。

核心关注点：

- 基于温度做债券组合配置
- 重点看每只基金在不同温度下的目标金额
- 通过温度矩阵调参，使输出贴近你的投资直觉
- 最终实现自动执行和自动提醒

## 已确认的策略口径

### 1. 从持仓驱动转向总额驱动

旧思路是“当前持仓 x 调整比例”，这会导致零持仓无法启动。

现在核心方向是：

- 先算总资产基准 `totalValueBasis`
- 再算每只基金的目标权重
- 最后得到目标金额和调整额

### 2. 缓冲池机制

组合分成三类资产角色：

- `buffer`：缓冲池
- `active`：主动债
- `offensive`：进攻池

进攻池资金变化最终要和缓冲池对冲，保证组合总额前后保持动态平衡。

### 3. 特殊资产规则

`兴全稳泰债券A` 是有特殊处理的：

- 它是主动债
- 它有单独的保护线
- 在低温区会有更高优先级
- 现在又支持按温度锚点配置目标权重

### 4. 调参方式

当前已经支持两层调参：

1. 全局公式调参
2. 个别基金按温度锚点配置目标权重

锚点字段现在是：

- `temperatureTargetWeights`

注意：

- 这个字段是权重，不是绝对金额
- 程序会用 `totalValueBasis x targetWeight` 算出最终目标金额
- 这样总资产基准变化时，曲线仍然成立

## 当前沟通中最重要的自动化目标

最核心的自动化目标已经明确：

- 通过飞书群里的自定义机器人 webhook 收到提醒
- 不再推进飞书私聊版
- 自动化重点放在稳定抓取、稳定定时、稳定通知

如果未来开新窗口，这一条必须优先继承，不要再把目标改回“飞书私聊”。
